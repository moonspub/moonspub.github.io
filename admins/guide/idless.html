IDless players

There is a flaw in Source games that allows players to connect to servers without being authenticated, which in turns means they have no Steam ID. This can make it challenging to deal with these players. Our server has some tools at your disposal to deal with these players.

identifying them

IDless players:

  • Cannot be targeted using normal commands like sm_ban/sm_kick
  • Do not have a connection message in the server's logging channel
  • Will show STEAM_ID_STOP_IGNORING_RETVALS for Steam IDs in other parts of the log
  • Cannot view profile by right-clicking on the scoreboard

dealing with them

We're using a custom plugin that will automatically disconnect these players after a certain duration after initial connection, and prevent them from spraying or using text chat. If for some reason this measure is bypassed (for example if they went IDless mid-game) you can use the sm_kickidless command to kick all currently IDless players.

It's worth mentioning that some players may be temporarily IDless after connecting or during Steam outages. Try to use the command sparingly as these players may also end up kicked. Also try to use it some time after a map change so all innocent players are authenticated.
